## Prerequisites

Before installing RigidUI components, ensure your project has the following setup:

<Steps>
  <Step title="Tailwind CSS">
    RigidUI components use Tailwind CSS for styling. Install and configure Tailwind CSS in your project:
    
    <CodeGroup>
    ```bash npm
    npm install -D tailwindcss postcss autoprefixer
    npx tailwindcss init -p
    ```
    
    ```bash yarn
    yarn add -D tailwindcss postcss autoprefixer
    npx tailwindcss init -p
    ```
    
    ```bash pnpm
    pnpm add -D tailwindcss postcss autoprefixer
    npx tailwindcss init -p
    ```
    </CodeGroup>
    
    Configure your `tailwind.config.js` to include your component paths:
    
    ```js tailwind.config.js
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      content: [
        "./app/**/*.{js,ts,jsx,tsx,mdx}",
        "./components/**/*.{js,ts,jsx,tsx,mdx}",
      ],
      theme: {
        extend: {},
      },
      plugins: [],
    }
    ```
  </Step>
  
  <Step title="shadcn/ui">
    RigidUI is built on top of shadcn/ui. Initialize shadcn/ui in your project:
    
    <CodeGroup>
    ```bash npm
    npx shadcn@latest init
    ```
    
    ```bash yarn
    npx shadcn@latest init
    ```
    
    ```bash pnpm
    npx shadcn@latest init
    ```
    </CodeGroup>
    
    Follow the prompts to configure your project. This will set up:
    - Component directory structure
    - Tailwind CSS configuration
    - CSS variables for theming
    - Utility functions
  </Step>
  
  <Step title="TypeScript">
    RigidUI components are built with TypeScript. Ensure your project has TypeScript configured:
    
    <CodeGroup>
    ```bash npm
    npm install -D typescript @types/node @types/react @types/react-dom
    npx tsc --init
    ```
    
    ```bash yarn
    yarn add -D typescript @types/node @types/react @types/react-dom
    npx tsc --init
    ```
    
    ```bash pnpm
    pnpm add -D typescript @types/node @types/react @types/react-dom
    npx tsc --init
    ```
    </CodeGroup>
  </Step>
</Steps>

<Note>
  If you're starting a new project, consider using Next.js with TypeScript and Tailwind CSS pre-configured:
  
  ```bash
  npx create-next-app@latest my-app --typescript --tailwind --app
  ```
</Note>

## Installing Components

Once your project meets the prerequisites, you can install any RigidUI component using the shadcn CLI:

### Basic Installation

Use the `npx shadcn@latest add` command with the component's registry URL:

```bash
npx shadcn@latest add https://rigidui.com/r/[component-name].json
```

<Tip>
  Replace `[component-name]` with the actual component name, such as `data-reveal`, `text-shift`, or `smart-form`.
</Tip>

## What Gets Installed?

When you install a RigidUI component:

1. **Component Files** - Added to your `components/ui` directory (or your configured components directory)
2. **Dependencies** - Any required npm packages are automatically installed
3. **Registry Dependencies** - Required shadcn/ui components are automatically added
4. **Utilities** - Helper functions and utilities are added if needed

<Warning>
  Components are copied to your project, not installed as npm dependencies. This gives you complete control but means updates require manually re-installing components.
</Warning>

## Verifying Installation

After installing a component, you should see:

- Component file(s) in your `components/ui` directory
- New dependencies in your `package.json` (if the component requires them)
- TypeScript definitions working in your editor

## Troubleshooting

<AccordionGroup>
  <Accordion title="Component not found error">
    Ensure you've initialized shadcn/ui with `npx shadcn@latest init` and that your components directory is correctly configured in `components.json`.
  </Accordion>
  
  <Accordion title="TypeScript errors after installation">
    Make sure all peer dependencies are installed. Run `npm install` (or `yarn`/`pnpm`) to ensure all dependencies are up to date.
  </Accordion>
  
  <Accordion title="Styling not working">
    Verify that:
    - Tailwind CSS is properly configured
    - Your CSS files include the Tailwind directives
    - The `tailwind.config.js` content array includes your component paths
  </Accordion>
  
  <Accordion title="Motion/animation errors">
    Some components use the `motion` library. Install it if you see errors:
    
    ```bash
    npm install motion
    ```
  </Accordion>
</AccordionGroup>
