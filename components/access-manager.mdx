## Overview

The Access Manager provides a comprehensive authentication and authorization system for React applications. It includes role-based access control (RBAC), permission management, session validation, and multiple components for controlling UI visibility based on user permissions.

## Installation

```bash
npx shadcn@latest add https://rigidui.com/r/access-manager.json
```

## Components

### AccessManagerProvider

The root provider component that manages authentication state and provides access control context to your application.

### AccessGate

Conditionally renders children based on user roles, permissions, and session validity.

### SessionGuard

Protects routes and components by ensuring the user has a valid session.

### ConditionalWrapper

Wraps children with different components based on user permissions.

## Usage Examples

### Basic Setup

```tsx
import { AccessManagerProvider } from '@/components/ui/access-manager'

const userSession = {
  id: '123',
  name: 'John Doe',
  email: 'john@example.com',
  roles: ['admin', 'editor'],
  permissions: ['read', 'write', 'delete'],
  isActive: true,
  expiresAt: new Date(Date.now() + 3600000) // 1 hour from now
}

const rolePermissionMap = {
  admin: ['read', 'write', 'delete', 'manage_users'],
  editor: ['read', 'write'],
  viewer: ['read']
}

function App() {
  return (
    <AccessManagerProvider 
      user={userSession} 
      rolePermissionMap={rolePermissionMap}
    >
      <YourApp />
    </AccessManagerProvider>
  )
}
```

### Role-Based Access Control

```tsx
import { AccessGate } from '@/components/ui/access-manager'

function AdminPanel() {
  return (
    <AccessGate roles="admin">
      <div>
        <h2>Admin Panel</h2>
        <p>Only administrators can see this content</p>
      </div>
    </AccessGate>
  )
}
```

### Permission-Based Access Control

```tsx
import { AccessGate } from '@/components/ui/access-manager'

function DeleteButton() {
  return (
    <AccessGate 
      permissions="delete"
      mode="disable"
      fallback={<Button disabled>Delete (No Permission)</Button>}
    >
      <Button variant="destructive">Delete</Button>
    </AccessGate>
  )
}
```

### Multiple Roles with Fallback

```tsx
import { AccessGate } from '@/components/ui/access-manager'

function ContentEditor() {
  return (
    <AccessGate 
      roles={['admin', 'editor']}
      fallback={
        <div className="text-muted-foreground">
          You need editor or admin role to access this feature
        </div>
      }
    >
      <RichTextEditor />
    </AccessGate>
  )
}
```

### Session Protection

```tsx
import { SessionGuard } from '@/components/ui/access-manager'

function ProtectedRoute() {
  return (
    <SessionGuard 
      redirectTo="/login"
      onSessionExpired={() => {
        console.log('Session expired, redirecting...')
      }}
    >
      <Dashboard />
    </SessionGuard>
  )
}
```

### Using the Hook

```tsx
import { useAccessManager } from '@/components/ui/access-manager'

function ProfileSettings() {
  const { user, hasPermission, isSessionExpiring } = useAccessManager()
  
  const canEditProfile = hasPermission('edit_profile')
  const sessionExpiringSoon = isSessionExpiring(10) // 10 minutes threshold
  
  return (
    <div>
      <h2>Welcome, {user?.name}</h2>
      {sessionExpiringSoon && (
        <Alert>
          Your session will expire soon. Please save your work.
        </Alert>
      )}
      {canEditProfile && <EditProfileButton />}
    </div>
  )
}
```

### Conditional Wrapper

```tsx
import { ConditionalWrapper } from '@/components/ui/access-manager'

function FeatureCard({ feature }) {
  return (
    <ConditionalWrapper
      permissions="premium"
      wrapper={(children) => (
        <Card className="border-gold">
          <Badge>Premium</Badge>
          {children}
        </Card>
      )}
      fallbackWrapper={(children) => (
        <Card className="opacity-50">
          <Badge>Upgrade Required</Badge>
          {children}
        </Card>
      )}
    >
      <CardContent>{feature.description}</CardContent>
    </ConditionalWrapper>
  )
}
```

## Hooks

### useAccessManager()

Returns the full access manager context:

- `user` - Current user session
- `rolePermissionMap` - Role to permission mapping
- `hasRole(role)` - Check if user has a specific role
- `hasPermission(permission)` - Check if user has a specific permission
- `hasAnyRole(roles)` - Check if user has any of the specified roles
- `hasAllRoles(roles)` - Check if user has all specified roles
- `hasAnyPermission(permissions)` - Check if user has any of the specified permissions
- `hasAllPermissions(permissions)` - Check if user has all specified permissions
- `isSessionValid()` - Check if the current session is valid
- `getSessionTimeRemaining()` - Get milliseconds until session expiration
- `isSessionExpiring(thresholdMinutes)` - Check if session expires within threshold

### usePermissionCheck()

Returns a boolean indicating if the user meets the specified criteria:

```tsx
const hasAccess = usePermissionCheck({
  roles: ['admin', 'editor'],
  permissions: 'write',
  requireAllRoles: false,
  requireAllPermissions: true,
  requireValidSession: true
})
```

<Note>
The Access Manager automatically validates session expiration and activity status. Set `expiresAt` on the user session to enable automatic session expiration checks.
</Note>

<Warning>
Always validate permissions on the server side as well. Client-side access control should be used for UI/UX purposes only, not as a security measure.
</Warning>

<Tip>
Use the `rolePermissionMap` to centralize permission management. This allows you to change permissions for a role without updating every component.
</Tip>
