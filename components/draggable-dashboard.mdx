## Overview

The Draggable Dashboard component enables users to create customizable dashboard layouts by dragging and dropping widgets. It includes automatic layout persistence, responsive grid systems, and lock/unlock functionality to prevent accidental changes.

## Installation

```bash
npx shadcn@latest add https://rigidui.com/r/draggable-dashboard.json
```

## Dependencies

This component requires `@dnd-kit` packages:

```bash
npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
```

## Features

- Drag-and-drop widget reordering
- Responsive grid layout (1-12 columns)
- Lock/unlock layout editing
- Persistent layout (localStorage)
- Widget spanning (multi-column/row)
- Keyboard navigation support
- Smooth animations and transitions

## Components

### DraggableDashboard

Main container component that manages the drag-and-drop grid.

### DraggableWrapper

Wrapper component for individual dashboard widgets.

## API Reference

### DraggableDashboard Props

<ParamField path="children" type="ReactNode" required>
  Dashboard widgets wrapped in DraggableWrapper components
</ParamField>

<ParamField path="className" type="string">
  Additional CSS classes for the dashboard container
</ParamField>

<ParamField path="showLockToggle" type="boolean" default="true">
  Whether to show the lock/unlock toggle control
</ParamField>

<ParamField path="showHandles" type="boolean" default="true">
  Whether to show drag handles on widgets
</ParamField>

<ParamField path="gridCols" type="number" default="3">
  Number of grid columns (1-12) on large screens
</ParamField>

<ParamField path="gap" type="number" default="6">
  Gap size between grid items (1-12, corresponds to Tailwind spacing)
</ParamField>

<ParamField path="defaultLocked" type="boolean" default="false">
  Whether the dashboard is locked by default
</ParamField>

<ParamField path="onOrderChange" type="(newOrder: string[]) => void">
  Callback fired when widget order changes
</ParamField>

<ParamField path="persistenceKey" type="string" default="'draggable-dashboard-order'">
  localStorage key for persisting widget order
</ParamField>

### DraggableWrapper Props

<ParamField path="id" type="string" required>
  Unique identifier for the widget
</ParamField>

<ParamField path="children" type="ReactNode" required>
  Widget content
</ParamField>

<ParamField path="gridSize" type="{ cols: number, rows: number }" default="{ cols: 1, rows: 1 }">
  Number of grid columns and rows the widget should span
</ParamField>

<ParamField path="className" type="string">
  Additional CSS classes for the wrapper
</ParamField>

<ParamField path="isLocked" type="boolean">
  Whether this specific widget is locked (managed by parent)
</ParamField>

<ParamField path="showHandle" type="boolean">
  Whether to show the drag handle (managed by parent)
</ParamField>

<ParamField path="availableCols" type="number">
  Available columns based on screen size (managed by parent)
</ParamField>

### Custom Grid Layout

```tsx

function CustomDashboard() {
  return (

          </CardContent>
```

### With Order Change Callback

```tsx
import DraggableDashboard, { DraggableWrapper } from '@/components/ui/draggable-dashboard'
import { useState } from 'react'

function AnalyticsDashboard() {
  const [widgetOrder, setWidgetOrder] = useState<string[]>([])

  const handleOrderChange = async (newOrder: string[]) => {
    setWidgetOrder(newOrder)
    
    // Save to backend
    await fetch('/api/dashboard/layout', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ order: newOrder })
    })
  }

  return (
    <div>
      <p className="mb-4">Current order: {widgetOrder.join(', ')}</p>
      <DraggableDashboard onOrderChange={handleOrderChange}>
        <DraggableWrapper id="metrics">
          <MetricsWidget />
        </DraggableWrapper>
        <DraggableWrapper id="chart">
          <ChartWidget />
        </DraggableWrapper>
        <DraggableWrapper id="table">
          <TableWidget />
        </DraggableWrapper>
      </DraggableDashboard>
    </div>
  )
}
```

## Responsive Behavior

- **Large screens (≥1024px)**: Uses specified `gridCols` value
- **Tablet (768px - 1023px)**: 2 columns
- **Mobile (&lt;768px)**: 1 column

## Keyboard Support

- **Space/Enter**: Activate drag mode
- **Arrow keys**: Move widget while dragging
- **Space/Enter**: Drop widget
- **Escape**: Cancel drag

<Note>
Widget layouts are automatically saved to localStorage and restored on page reload. Use different `persistenceKey` values for different dashboards or users.
</Note>

<Tip>
For the best user experience:
- Use the lock toggle to prevent accidental changes in production
- Keep widget heights consistent within rows for better alignment
- Test your dashboard on different screen sizes
</Tip>

<Warning>
Ensure each `DraggableWrapper` has a unique `id` prop. Duplicate IDs will cause unexpected behavior in the drag-and-drop functionality.
</Warning>
