<ParamField path="primaryText" type="string" required>

  Custom easing curve array (cubic bezier values)
</ParamField>

## Animation Details

### Primary Text Animation (on hover)

```tsx
{
  y: "-100%",    // Slides up out of view
  opacity: 0.8   // Fades slightly
}
```

### Secondary Text Animation (on hover)

```tsx
{
  y: "0%",       // Slides into view from below
  opacity: 1     // Fully visible
}
```

### Letter Stagger Effect

Each letter animates with a delay calculated as:

```ts
delay = index * staggerDelay
```

For example, with default `staggerDelay` of 0.05s:
- Letter 0: 0s delay
- Letter 1: 0.05s delay
- Letter 2: 0.10s delay
- Letter 3: 0.15s delay

<Callout type="info">
  The stagger effect creates a wave-like animation as letters shift sequentially. Adjust `staggerDelay` to control the wave speed.
</Callout>

## Easing Curves

### Default Easing

The default easing curve `[0.165, 0.84, 0.44, 1]` provides a smooth, natural motion.

### Custom Easing Examples

```tsx
// Bouncy effect
easing={[0.68, -0.55, 0.265, 1.55]}

// Quick snap
easing={[0.95, 0.05, 0.795, 0.035]}

// Elastic
easing={[0.34, 1.56, 0.64, 1]}

// Smooth ease-in-out
easing={[0.45, 0, 0.55, 1]}
```

<Callout type="tip">
  Use [cubic-bezier.com](https://cubic-bezier.com) to visualize and create custom easing curves.
</Callout>

## Performance Considerations

<Callout>
  The component splits text into individual letters, creating separate motion components for each character. For very long text, consider using shorter phrases for optimal performance.
</Callout>

### Best Practices

1. **Keep text concise**: Works best with 1-15 characters
2. **Use uppercase**: Uppercase text often looks better with this effect
3. **Adjust stagger for length**: Shorter delays for longer words
4. **Test on mobile**: Ensure hover alternatives for touch devices

<Callout type="warning">
  On touch devices, the hover state is triggered on tap. Consider adding a tap-away behavior or auto-reset for better UX on mobile.
</Callout>
